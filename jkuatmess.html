Methodology
The full pipeline, grounded in the inventory models notes. Three models, one integrated system.

Phase 1 — Inventory classification

SKUs are segmented by perishability. This replaces the notes' one-size-fits-all EOQ with three class-specific policies. Ultra-perishables cannot use (Q,R) because they cannot be held across cycles; non-perishables don't need newsvendor because spoilage isn't binding.

Phase 2 — Learned demand distributions

The notes assume demand is either deterministic or drawn from a fixed distribution. In practice demand is non-stationary. We replace the assumed distribution with a conditional one learned from POS data via quantile regression.

Phase 3 — Probabilistic policy engine

Each class applies its appropriate model. Newsvendor for Class A (single-period, critical ratio). (Q,R) with safety stock for Class C (continuous review, service-level target). Rolling horizon for Class B (receding horizon re-optimisation).

Phase 4 — System dynamics simulation

No single-item policy can see feedback loops between items, suppliers, and manager behaviour. We simulate the policy inside a feedback-rich environment before committing — capturing bullwhip, oscillation, and cascading stockouts.
